四大组件
Intent
1. 什么是Intent
Intent是Android中用于组件间通信的机制，负责传递操作、数据和相关信息，应用于内部和应用间的交互与调用。
2. 启动三大组件
1.Activity:startActivity(Intent intent);//过时方法 推荐：registerForActivityResultstartActivityForResult(Intent intent,int requestCode);
2.Service:startService(Intent intent);bindService(Intent intent,ServiceConnection conn,int flags);
3.BroadcastReceiver:sendBroadcast(Intent intent);sendOrderedBroadcast(Intent intent,String receiverPermission);
3. Intent的构成
Action: 用来表现意图的行动
一个字符串变量，可以用来指定Intent要执行的动作类别。比如查看或选择，其对应着Intent Filter中的action标签。
<intent-filter> <action android:name="android.intent.action.MAIN" /> <category android:name="android.intent.category.LAUNCHER" /></intent-filter>

Data: 表示与动作要操纵的数据。
Intent中的data指的是Uri对象和数据的MIME类型，其对应着Intent Filter中的data标签。
1.Uri格式:<scheme>://<host>:<port>/<path>
oscheme: 协议类型（如 http、https、content等）
ohost: 主机名（如域名或IP地址）
oport: 端口号（可选）
opath: 资源路径
2.示例代码
1.查看图像文件// 创建一个Uri对象，指向要查看的图像文件val imageUri: Uri = Uri.parse("content://com.example.project:200/folder/subfolder/image.jpg")// 创建一个Intent来启动查看图像的活动
val viewImageIntent = Intent(Intent.ACTION_VIEW).apply {// 设置Uri和MIME类型setDataAndType(imageUri, "image/jpeg")}// 检查是否有应用可以处理此Intentif (viewImageIntent.resolveActivity(packageManager) != null) {// 启动活动startActivity(viewImageIntent)}
2.编辑文档文件// 创建一个Uri对象，指向要编辑的文档文件val documentUri: Uri = Uri.parse("content://com.example.project:200/folder/subfolder/document.txt")// 创建一个Intent来启动编辑文档的活动val editDocumentIntent = Intent(Intent.ACTION_EDIT).apply {// 设置Uri和MIME类型setDataAndType(documentUri, "text/plain")}// 检查是否有应用可以处理此Intentif (editDocumentIntent.resolveActivity(packageManager) != null) {// 启动活动startActivity(editDocumentIntent)} ```
Category: 用来表现动作的类别。
一个包含Intent额外信息的字符串，表示哪种类型的组件来处理这个Intent。任何数量的Category 描述都可以添加到Intent中，但是很多intent不需要category。
Category
描述
CATEGORY_DEFAULT
表示该Intent可以作为默认的启动意图处理。
CATEGORY_BROWSABLE
表示该Intent可以被浏览器或其他类似的应用处理。
CATEGORY_LAUNCHER
表示该Intent是一个启动器应用。
CATEGORY_SETTINGS
表示该Intent用于显示应用的设置界面。
CATEGORY_SHARE
表示该Intent用于分享内容，例如文本或图像。
示例代码
<intent-filter> <!-- 指定该Activity可以处理VIEW类型的Intent --> <action android:name="android.intent.action.VIEW" /> <!-- 表示该Intent可以作为默认的启动意图处理 --> <category android:name="android.intent.category.DEFAULT" /> <!-- 自定义的Category，允许其他应用通过该类别来调用此Activity --> <category android:name="com.baidu.category.TEST" />
<!-- 指定该Activity可以处理以http开头的URI --> <data android:scheme="http" /> </intent-filter>
<!---->
// 创建一个Uri对象，指向要分享的内容 val contentUri: Uri = Uri.parse("content://com.example.project:200/folder/subfolder/document.txt") // 创建一个Intent来分享内容，使用ACTION_SEND以便共享数据 val shareIntent = Intent(Intent.ACTION_SEND).apply { // 使用putExtra方法将要分享的Uri添加到Intent中 putExtra(Intent.EXTRA_STREAM, contentUri) // 设置分享的MIME类型，这里为文本文件 type = "text/plain" // 添加默认的Category，使其可作为默认选项被处理 addCategory(Intent.CATEGORY_DEFAULT) } // 检查系统中是否有可处理此Intent的应用 if (shareIntent.resolveActivity(packageManager) != null) { // 如果有，则启动分享活动，通过Chooser选择应用 startActivity(Intent.createChooser(shareIntent, "分享文件")) }
Type: 指定数据类型。
在Android中，Intent的数据类型通常根据其内容自动推断。但可以通过设置type属性强制指定一个明确的类型，避免自动推导。例如，可以通过setDataAndType方法同时设置数据的URI和MIME类型。示例代码
// 创建一个Intent对象 val intent = Intent().apply { // 设置数据的URI和MIME类型 setDataAndType(Uri.parse("baidu://www.baidu.com/news"), "image/jpeg") }
<!---->
<data #指定可处理的MIME类型为所有图片类型 android:mimeType="image/*" #指定URI的方案为"baidu"，表明此数据通过"baidu"协议传输 android:scheme="baidu" #指定URI的主机部分为"www.baidu.com"，表示仅处理来自此主机的请求 android:host="www.baidu.com" #指定URI的路径部分为"/news"，表示仅处理该路径下的请求 android:path="/news"/>
Component: 目的组件
在Android中，Component属性用于指定Intent的目标组件名称。当设置了这个属性后，系统将跳过其他匹配属性，直接根据这个属性启动对应的组件。
Extra: 扩展信息
在Android中，Intent可以携带额外的key-value数据。这些数据可以在目标组件（如Activity或Service）中使用。你可以通过调用putExtra()方法将数据添加到Intent中，或者通过创建一个Bundle对象来存储多个数据，然后使用putExtras()方法将其添加到Intent中。
字段
说明
类型
Intent.EXTRA_EMAIL
主要收件人的电子邮件地址。
String[]
Intent.EXTRA_CC
抄送收件人的电子邮件地址。
String[]
Intent.EXTRA_BCC
密送收件人的电子邮件地址。
String[]
Intent.EXTRA_SUBJECT
电子邮件的主题。
String
Intent.EXTRA_TEXT
电子邮件的正文内容。
String
Intent.EXTRA_STREAM
电子邮件中包含的附件URI。
Uri
Intent.EXTRA_TITLE
电子邮件对话框的标题（可选）。
示例代码
// 创建一个Intent来发送电子邮件 val emailIntent = Intent(Intent.ACTION_SENDTO).apply { data = Uri.parse("mailto:") // 仅允许邮件应用处理此Intent // 添加收件人、抄送、密送 putExtra(Intent.EXTRA_EMAIL, arrayOf("recipient@example.com")) // 主要收件人 putExtra(Intent.EXTRA_CC, arrayOf("cc@example.com")) // 抄送收件人 putExtra(Intent.EXTRA_BCC, arrayOf("bcc@example.com")) // 密送收件人 // 添加主题和正文 putExtra(Intent.EXTRA_SUBJECT, "Subject Here") // 邮件主题 putExtra(Intent.EXTRA_TEXT, "Body of the email") // 邮件正文 // 可选：添加附件 putExtra(Intent.EXTRA_STREAM, Uri.parse("file://path/to/attachment")) // 附件URI } // 确保有可用的邮件应用来处理此Intent if (emailIntent.resolveActivity(packageManager) != null) { startActivity(emailIntent) // 启动邮件应用 }
Flag: 期望这个意图的运行模式。
在Android中，Flag用于指示系统如何启动一个Activity。它们可以通过setFlags()或addFlags()方法添加到Intent中，以控制Activity的行为和启动模式。
Flag
说明
Intent.FLAG_ACTIVITY_NEW_T
ASK
启动一个新的任务，并在新的任务中启动Activity。
Intent.FLAG_ACTIVITY_CLEAR
_TOP
如果目标Activity已经在任务中，则将其上面的所有Activity移除，并重新启动目标Activity。
Intent.FLAG_ACTIVITY_SINGL
E_TOP
如果目标Activity已经在顶部，则不会创建新的实例，而是调用现

Intent.FLAG_ACTIVITY_CLEAR
_TASK

Intent.FLAG_ACTIVITY_REORD
ER_TO_FRONT

示例代码
// 创建一个Intent对象 val intent = Intent(this, SecondActivity::class.java).apply { // 添加Flags以控制Activity的启动方式 addFlags(Intent.FLAG_ACTIVITY_NEW_TASK) // 新建任务 addFlags(Intent.FLAG_ACTIVITY_CLEAR_TOP) // 清除上面的Activity } // 启动目标Activity startActivity(intent)
